# Product

## Register

product

## Users

Team AI Agent is built for whole software teams: founders, product owners, project managers, designers, developers, QA, DevOps, support, and operations leaders who want an AI company-like team to take a goal from brief to shipped output. Users work in a product/company context rather than a casual chat context. They need to describe an outcome once, watch the AI organization assign roles, produce artifacts, test results, deployments, and a final delivery report.

## Product Purpose

Team AI Agent is an autonomous AI company simulator and execution platform. The user gives a mission, then a full virtual dev company handles discovery, planning, design, engineering, QA, deployment, documentation, and reporting through specialized AI roles. Success means the system can finish substantial work end to end, explain what happened, preserve every artifact, and make progress visible through a strategy-game-style interface.

## Brand Personality

Strategic, capable, alive. The product should feel like a serious operating room for work, presented through a pixel/strategy-game company world. It should feel energetic and delightful without becoming childish. The AI team should feel organized, accountable, and busy in ways the user can inspect.

## Anti-references

Avoid generic chatbot interfaces, bland SaaS dashboards, toy-like idle games, childish pixel art, unreadable pixel fonts, purple-gradient AI branding, black-box automation, and fake productivity theater. The product should not look like a landing page pretending to be an app. It should not hide serious execution behind cute animation.

## Design Principles

1. Show the company working. Every autonomous action should have a visible owner, role, room, state, artifact, or trace.
2. Autonomy with accountability. The default mode is end-to-end autopilot, but the user can inspect decisions, tool calls, costs, risks, and outputs at any time.
3. Game layer explains work, it does not replace work. Pixel scenes make the system understandable and memorable; text, tables, logs, diffs, and reports remain first-class.
4. Roles have real responsibilities. CEO, PM, BA, designers, engineers, QA, DevOps, HR, finance, and support agents must produce different outputs and hand work to each other.
5. Trust beats spectacle. Motion, character activity, and status effects should communicate progress, blockers, review, test results, and deployment state.

## Accessibility & Inclusion

Target WCAG 2.2 AA. All statuses must be available through text and icons, not color or animation alone. Pixel visuals must have readable non-pixel UI labels. Provide reduced-motion mode, keyboard navigation, screen-reader-friendly logs, clear focus states, and high-contrast alternatives for long work sessions.
